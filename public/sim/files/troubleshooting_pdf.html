Password & Account

Verify identity; do not ask users to reveal passwords. Use these commands to check account status, reset or unlock.

Windows
PowerShell
# Check AD account status (AD module) Get-ADUser -Identity "jsmith" -Properties LockedOut,PasswordExpired,Enabled
# Unlock account and reset password Unlock-ADAccount -Identity "jsmith"; Set-ADAccountPassword -Identity "jsmith" -Reset -NewPassword (ConvertTo-SecureString "TempP@ssw0rd!" -AsPlainText -Force)
Linux
CLI
# Change local password (as admin) sudo passwd jsmith
# View failed auth attempts (systemd/journal) sudo journalctl -u sshd --since "1 hour ago" | tail -n 50
macOS
# Reset local account password (macOS) sudo dscl . -passwd /Users/jsmith "NewTempPass123!"

Network & DNS

Check connectivity, DNS resolution, and network routes.

Windows
# Basic network info & DNS cache ipconfig /all ipconfig /flushdns
# Test connectivity / trace ping -n 4 8.8.8.8 tracert example.com
Linux / macOS
# IP, route, DNS lookup ip a && ip route ping -c 4 8.8.8.8 dig +short example.com || nslookup example.com
# Trace route traceroute example.com # macOS/Linux (use 'tracepath' on some distros)

Service / Process

Restart, check status, or view logs for services.

Linux (systemd)
# Check service status, restart, view logs sudo systemctl status nginx sudo systemctl restart nginx sudo journalctl -u nginx -n 200 --no-pager
Windows
# Services via PowerShell Get-Service -Name wuauserv Restart-Service -Name "wuauserv" -Force Get-EventLog -LogName System -Newest 200 | Where-Object {$_.Source -like "*Service*"}
macOS
# macOS launchctl / process sudo launchctl list | grep myservice sudo launchctl kickstart -k system/com.example.service ps aux | grep -i myservice

Disk & Storage

Check disk space, inodes, and basic file system health.

# Disk usage, inodes, top directories df -hT df -i du -sh /var/* | sort -hr | head -n 20
# Windows disk space & chkdsk wmic logicaldisk get size,freespace,caption chkdsk C: /f
# macOS disk usage & repair diskutil list diskutil verifyVolume /Volumes/YourVolume

Logs & Diagnostics

Collect and tail logs to find recent errors.

# Tail log files sudo tail -n 200 /var/log/syslog sudo tail -F /var/log/nginx/error.log
# Windows Event logs (PowerShell) Get-EventLog -LogName Application -EntryType Error -Newest 100 Get-WinEvent -FilterHashtable @{LogName='System'; Level=2} -MaxEvents 200
# macOS logs log show --predicate 'eventMessage CONTAINS "error"' --last 1h

Authentication / SSO

Check identity provider status, token caches, and common SSO issues.

# Azure AD (MSOnline/AzureAD modules) — example: reset password (requires module + permissions) Set-MsolUserPassword -UserPrincipalName user@contoso.com -NewPassword "TempP@ssw0rd!" -ForceChangePassword $true
# Check ADFS service on Windows Server Get-Service adfssrv Get-EventLog -LogName AD FS/Admin -Newest 100
# Check IDP health (example HTTP check) curl -I https://idp.example.com/.well-known/openid-configuration

MFA (Authenticator & Push)

Common fixes: clock skew, re-enroll authenticator, use backup codes.

# Azure AD: revoke refresh tokens for a user (forces reauth) Revoke-AzureADUserAllRefreshToken -ObjectId
# TOTP clock skew: instruct user to re-sync app (Google Authenticator on Android) # Alternatively, re-provision by removing & re-adding the account.
Quick cheats & reminders
Safe verification

Ask for verification items (last 4 of phone, employee ID, home office) — never request the password itself.

Common troubleshooting flow
  1. Verify identity
  2. Confirm exact error + screenshot
  3. Try simple fixes (caps lock, different device)
  4. Check logs & account status
  5. Reset / escalate if compromise suspected
