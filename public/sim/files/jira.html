I've created a JIRA ticket for a regression impacting login session handling on the staging environment. This issue is causing users to be unexpectedly logged out after ~5 minutes of inactivity (previously 30 minutes).

Ticket summary
  • ID: PROJ-4521
  • Priority: Blocker
  • Assignee: @rachel.chen
  • Reporter: Alex Martin
Description

Sessions expire prematurely on staging after a recent auth cookie change (deploy 2025-10-05 02:14 UTC). Affected flows include SSO and direct login. Steps to reproduce and logs are attached.

Steps to reproduce
  1. Log in to staging.company.com using a valid user (SSO or local).
  2. Leave the browser idle for ~5 minutes (no activity).
  3. Attempt to navigate — the user is redirected to the login page and sees a 401 toast.
Observed

Immediate redirect to login and "Session expired" toast. Session cookie `acme_sess` shows `Max-Age` reduced to 300s in response headers.

Expected

Sessions remain active for 30 minutes of inactivity; refresh token flow should silently renew session.
