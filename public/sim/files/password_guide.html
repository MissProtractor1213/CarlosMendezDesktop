Password Troubleshooting Guide

Practical steps for help desks, admins, and users — Ready to print as PDF · Oct 5, 2025

Quick checklist — before escalating

Confirm username — verify user typed correct account (email vs local username).
Caps / Num lock — common cause on laptops and external keyboards.
Password manager — check saved credentials or offer to copy/paste from manager.
Recent password changes — user may be using an old password.
Account lockouts — confirm last failed-attempt time and lockout policy.

Step-by-step troubleshooting (fast)

  1. Verify identity — ask standard help-desk questions per policy (do not ask for full password).
  2. Confirm exact error message — copy/paste or screenshot of the error text.
  3. Try known-working keyboard — ask them to type password into a text field (not the password field) to confirm characters shown.
  4. Reset if needed — follow organization password reset flow (self-service portal or admin reset). Force temporary password and require user to change at next sign-in when possible.
  5. Check MFA — if MFA is enabled, verify device and time-sync; check for blocked push notifications or changed phone numbers.
  6. Account status — check AD/IDP for disabled/expired accounts, password expired flags, or recent admin changes.

Troubleshooting flow (quick)

1) Can user type password visibly? → If yes, check characters and caps lock. If no, try alternate keyboard or device.

2) Does account accept temporary password? → If yes, force reset and confirm MFA. If no, check account status (disabled/expired/locked).

3) MFA failing? → Resync, clear cached tokens, or use recovery codes. If not resolved, escalate to IDP team.

Secure practices & guidance for users

  • Create unique passwords — recommend a phrase of 12+ characters or use a password manager.
  • Enable MFA — use authenticator app or hardware key where possible rather than SMS.
  • Never share passwords — never ask for a full password during troubleshooting; request verification questions instead.
  • Use passkeys / SSO when available — reduces password-related support load.

When to escalate

  • Multiple accounts failing across many users (possible outage).
  • Evidence of compromise (password spray, unusual lockouts).
  • MFA device removal or SIM swap suspected.
  • Cannot unlock or reset via normal admin tools.

Sample help-desk script

Agent: "Hi — I can help with this. I won't ask you to tell me your password. Can you confirm your username and the exact error message?"

User: "It says 'account locked'."

Agent: "Thanks. I'll check your account status and either unlock it or initiate a reset. Please confirm the last 4 digits of your phone number on file."

Examples: Recovery codes & MFA

Authenticator app: To fix time-based failures, open app settings → Sync time correction (Android) or re-scan QR code.

Backup codes: Store securely (password manager) — they are single-use and replace MFA when phone is unavailable.
